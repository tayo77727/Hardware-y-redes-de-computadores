Introducción al proyecto

Este sitio educativo presenta conceptos clave de hardware y redes de forma clara y práctica. Está diseñado para aprender lo esencial, interactuar con el modelo OSI y reforzar el conocimiento con un quiz y un glosario.

¿Qué es hardware y redes?

El hardware es el conjunto de componentes físicos de un sistema computacional: desde la CPU y la RAM hasta periféricos y dispositivos de almacenamiento. Las redes conectan equipos para compartir datos y recursos mediante dispositivos como switches y routers, empleando protocolos y direcciones IP.

Objetivo del sitio

Comprender los componentes fundamentales del hardware, los elementos básicos de las redes y el modelo OSI, con ejemplos sencillos y actividades de verificación del aprendizaje.

Hardware

CPU

La Unidad Central de Procesamiento ejecuta instrucciones y coordina el funcionamiento del sistema. Su rendimiento depende de la frecuencia, núcleos y arquitectura. Es el "cerebro" del computador.

RAM

La memoria de acceso aleatorio almacena temporalmente datos que la CPU necesita de forma rápida. Más RAM permite manejar más tareas simultáneas sin recurrir a almacenamiento más lento.

Almacenamiento

Guarda datos de forma permanente. Los SSD ofrecen mayores velocidades que los HDD tradicionales. El tipo y la interfaz (SATA, NVMe) impactan en el rendimiento general.

Placa base

Integra y conecta todos los componentes. Define compatibilidades (socket de CPU, tipo de RAM, puertos) y distribuye energía y comunicaciones entre dispositivos.

Fuente de poder

Suministra energía estable a todos los componentes. Su calidad y capacidad afectan la estabilidad del sistema; se mide en watts y eficiencia.

Periféricos

Dispositivos externos de entrada y salida como teclado, ratón, monitor, impresora y cámaras. Amplían las capacidades de interacción del equipo.

Redes

Switch

Conecta múltiples dispositivos en una red local y reenvía tramas según direcciones MAC, optimizando el tráfico dentro del segmento.

Router

Interconecta redes distintas y dirige paquetes según rutas e IP. Suele proporcionar funciones de seguridad y NAT para acceso a internet.

IP

La dirección IP identifica de forma lógica a cada dispositivo. Puede ser IPv4 (p. ej., 192.168.1.10) o IPv6. Permite el enrutamiento correcto.

DHCP

Protocolo que asigna automáticamente direcciones IP, máscara, puerta de enlace y DNS a los clientes, simplificando la administración.

DNS

Convierte nombres de dominio legibles (ej. ejemplo.com) en direcciones IP numéricas. Facilita el acceso a servicios sin memorizar números.

Subred (ejemplo)

En 192.168.1.0/24 hay 256 direcciones posibles (0–255). Usando /26 se crean 4 subredes con 64 direcciones cada una, organizando mejor el tráfico.

Topologías de red

Bus

Todos los dispositivos comparten un único canal. Es simple y económica, pero sufre colisiones y no escala bien.

Estrella

Cada dispositivo se conecta a un nodo central (switch). Fácil de administrar y aislar fallos, común en redes LAN modernas.

Anillo

Los nodos forman un círculo y los datos viajan en una dirección. Tiene tolerancia limitada a fallos si no hay mecanismos de redundancia.

Malla

Múltiples conexiones entre nodos proporcionan alta redundancia y rutas alternativas, mejorando disponibilidad y resiliencia.

Glosario

CPU

Unidad Central de Procesamiento; ejecuta instrucciones y coordina el sistema.

RAM

Memoria temporal de acceso rápido utilizada por la CPU para tareas activas.

SSD

Dispositivo de almacenamiento sólido, más rápido que los discos duros tradicionales.

Switch

Dispositivo que reenvía tramas dentro de una LAN utilizando direcciones MAC.

Router

Interconecta redes distintas y decide rutas para los paquetes según IP.

DHCP

Protocolo que asigna de forma automática configuraciones IP a los clientes.

DNS

Servicio que traduce nombres de dominio en direcciones IP numéricas.

OSI

Modelo de referencia con 7 capas que guía el diseño y operación de redes.

Subred

Segmento de una red IP con máscara que delimita el rango de direcciones.
